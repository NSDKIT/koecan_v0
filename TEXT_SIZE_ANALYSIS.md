## レスポンシブ対応のパターン

### パターン1: モバイル小 → デスクトップ大
- `text-xs sm:text-sm`: タブナビゲーション、キャリア相談ボタン上部
- `text-sm sm:text-lg`: ポイント/パーソナリティタイプラベル
- `text-sm sm:text-base`: モーダルタブボタン、キャリア相談ボタン
- `text-base sm:text-lg`: キャリア相談ボタンメイン
- `text-3xl sm:text-5xl`: ポイント/パーソナリティタイプ数値
- `text-2xl sm:text-4xl`: 企業名

## サイズの階層構造

### 最大サイズ (text-5xl / 48px)
- ポイント数値、パーソナリティタイプ数値（デスクトップ）

### 大見出し (text-4xl / 36px)
- 企業名（デスクトップ）

### 見出し1 (text-2xl / 24px)
- アプリタイトル、セクションタイトル、企業名（モバイル）

### 見出し2 (text-xl / 20px)
- カードタイトル、価値観詳細セクション

### 見出し3 (text-lg / 18px)
- 質問タイトル、ポイントバッジ、空状態メッセージ

### 本文 (text-base / 16px)
- ボタンテキスト、メインコンテンツ

### 小テキスト (text-sm / 14px)
- メタ情報、バッジ、説明文、リンクボタン

### 最小テキスト (text-xs / 12px)
- ユーザーID、LINE ID、選択数バッジ

## 推奨事項

1. **一貫性の確保**: 同じ役割の要素には同じサイズを使用
2. **階層の明確化**: 見出しの階層を明確にする
3. **レスポンシブ対応**: 重要な要素はモバイルとデスクトップでサイズを調整
4. **可読性**: 小さすぎるテキスト（text-xs）は補助情報に限定
